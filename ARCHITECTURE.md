# Auvra AI Architecture

The architecture seamlessly weaves Auvra Agentic Payments, Chainlink CRE, Tenderly, and World ID into a unified risk-firewall.

## Architecture Diagram

```mermaid
sequenceDiagram
    participant AI as Auvra AI Sentinel
    participant CRE as Chainlink CRE DON (TS Workflow)
    participant Tenderly as Tenderly Virtual TestNet
    participant Admin as Human Protocol Admin (thirdweb app)
    participant WID as World ID
    participant Chain as Target DeFi Protocol (Mainnet)

    %% Autonomy Phase
    AI->>AI: Detects Anomalous On-chain Activity
    Note over AI,CRE: Auvra x402 Payment
    AI->>CRE: Triggers Workflow (Passkey Session Signed)
    
    %% Evaluation Phase
    CRE->>CRE: Validates Risk Parameters (Confidential Compute)
    
    %% Simulation Phase
    CRE->>Tenderly: HTTPClient: POST /simulate (Pause Protocol)
    Tenderly-->>CRE: Simulation Success (No Revert)
    
    %% Notification Phase
    CRE->>Admin: Alerts Admin with Tenderly Simulation Link
    
    %% Human-in-the-Loop Phase
    Admin->>WID: Requests BIOMETRIC authentication
    WID-->>Admin: Generates ZK Proof of Humanness
    Admin->>CRE: Submits World ID ZKP + Transaction Approval
    
    %% Execution Phase
    CRE->>CRE: Validates World ID Proof off-chain
    CRE->>Chain: EVMClient: Execute pause() on DeFi Protocol
    Chain-->>CRE: Execution Confirmed
```

### Components Breakdown

1. **AI Risk Sentinel (Auvra Agent)**:
   - Node.js background worker running risk heuristic models.
   - Initialized via `@veridex/agentic-payments` SDK.
   - Possesses a 24-hour time-limited, fund-limited passkey generated by a human founder.

2. **Chainlink CRE (The Orchestrator)**:
   - Built via the `@chainlink/cre-sdk` in TypeScript.
   - Contains a `handler()` that maps an `http.Trigger` to the core logic callback.
   - Interacts with both `HTTPClient` and `EVMClient` concurrently.
   
3. **Tenderly API**:
   - Accepts the proposed `data` payload and simulates exactly what the result will be without spending gas or risking mainnet assets.

4. **World ID Integration**:
   - Embedded using the `IDKit` interface into a lightweight React frontend deployed alongside the protocol.
   - Guarantees the actor signing the final protocol change is a real human (Sybil and AI-resistant).
